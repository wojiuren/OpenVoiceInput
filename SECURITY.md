# Security Policy

OpenVoiceInput-MVP handles microphone input and clipboard output, so privacy-sensitive reports are especially important.

## Supported Versions

The project is currently in test-build status. Security fixes target the latest public test release.

## Reporting A Vulnerability

Please open a private report through GitHub security advisories if available, or contact the maintainer directly before posting sensitive details publicly.

Helpful details include:

- Operating system and Python version.
- OpenVoiceInput-MVP version or commit.
- Whether the issue involves microphone capture, clipboard output, API post-processing, local files, or packaging.
- Minimal steps to reproduce without sharing private audio or API keys.

## Privacy-Sensitive Data

Do not attach real recordings, API keys, personal paths, or private logs to public issues. If a reproduction needs sample audio, use synthetic or non-sensitive audio.
